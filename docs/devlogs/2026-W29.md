## 关键决策

- 应用目录模式下只允许一个应用真相源，不合并假数据与可写目录。
- v1 采用不可转移的所有者作用域；OIDC 成员关系缺失时在存储库查询前失败关闭。
- 归档为不可恢复的软归档；历史证据继续可读，新写入和 Web 新调用交接被阻断。
- 既有应用草案和发布候选 schema 暂不静默升级，继续通过 `base_application_updated_at` 表达基线漂移。
- API 密钥生命周期必须和 Gateway 真实验证同批闭环；开发测试态密钥与 provider credential、生产 API 密钥严格分离。
- 原始密钥不可恢复；需要换密时采用“先签发新密钥并验证，再吊销旧密钥”，v1 不提供原地轮换或反吊销。
