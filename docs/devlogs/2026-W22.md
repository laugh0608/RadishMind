## 关键决策

- provider capability matrix 只证明 provider 能力声明和 discovery 口径可检查，不证明 provider health、optional live health 或 production readiness。
- 当前不引入第二套 provider truth；矩阵必须复用 `services/runtime/provider_registry.py`。
- provider health smoke v1 只证明 mock runtime 与离线配置态可检查；optional live health 必须显式手动运行，失败只能作为 provider health signal，不得写成 production outage。
- provider selection policy v1 只固定选择身份、负向分类和 no implicit fallback，不声明 retry/fallback execution、live timeout probe 或 production readiness 已实现。
- provider retry/fallback policy v1 只固定当前审计策略和失败路径行为：调用方管理 retry，平台 fallback disabled；它不代表自动 retry、provider failover、backoff、budget 或 latency policy 已实现。
- `provider-runtime-docs-refresh` 只固定文档和治理口径，说明 Provider Runtime & Health v1 进入 close candidate；它不代表 optional live health、retry/fallback execution、production secret backend、tool executor、confirmation、writeback、replay 或 production readiness 已完成。
- 本地 container smoke 通过后，只能把 `docker_local` mock provider 容器路径推进到可审计证据状态；测试环境 smoke、production preflight、镜像发布、production secret backend、正式 auth / CORS、process supervisor 和 production ready 仍不满足。
- `production-secret-backend-contract` 只能证明 production secret backend 的接口和停止线可检查；真实 production secret backend、secret rotation policy、production secret audit store 和云 secret 服务集成仍为 `not_satisfied`。
- `production-secret-backend-implementation-readiness` 只能证明下一步实现前置条件已写清；后续如继续推进，应从 `secret-ref-schema-and-fixtures` 开始，而不是直接接云服务或写真实 secret。
