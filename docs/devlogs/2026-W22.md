## 关键决策

- provider capability matrix 只证明 provider 能力声明和 discovery 口径可检查，不证明 provider health、optional live health 或 production readiness。
- 当前不引入第二套 provider truth；矩阵必须复用 `services/runtime/provider_registry.py`。
- provider health smoke v1 只证明 mock runtime 与离线配置态可检查；optional live health 必须显式手动运行，失败只能作为 provider health signal，不得写成 production outage。
- provider selection policy v1 只固定选择身份、负向分类和 no implicit fallback，不声明 retry/fallback policy、live timeout probe 或 production readiness 已实现。
- `provider-runtime-docs-refresh` 只固定文档和治理口径，说明 Provider Runtime & Health v1 进入 close candidate；它不代表 optional live health、真实 retry/fallback、production secret backend、tool executor、confirmation、writeback、replay 或 production readiness 已完成。
