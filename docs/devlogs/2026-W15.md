## 关键决策

- `recommended summary` 的 same-sample 与 cross-sample 两路状态继续显式保留，但路径解析、默认占位和执行结果回写统一通过共享 helper 收口
- 临时目录回归脚本继续只表达“场景差异”和“断言差异”，不再各自重复拼接一整套 real batch 编排命令
- 对于“基于真实 record 派生、但仍以本地 manual fixture 入仓”的负例，不再只靠 `notes` 记录来源，而是强制补 `source_candidate_response_record` 结构化反链，并生成独立 index 做可审计收口
- 后续 real-derived negative 扩样优先围绕“真实 bad record 原有失败面 + 一层额外越界漂移”的组合构造，而不是重新回到泛化的 simulated negative 堆叠
- 当同一条 source record 开始出现多个 real-derived 变体时，索引侧优先补轻量 pattern 分组，而不是先引入更重的专用结构字段
- `violation_groups` 继续保持“按完整 `expected_candidate_violations` 字符串分组”的朴素口径，不为了压平 confirmation 型 action title 差异而额外引入新的归一化层；跨 source 的复用审计优先依赖 `pattern_groups`

## 风险与未完成项

- 当前虽然已补首批 real-record-derived 的 `issue / proposed_actions / requires_confirmation` 组合失稳，但真实 provider 原始坏输出仍以 `read_only_check` 缺失、`answer` 缺失和 citation/source 脱节为主，后续还需继续扩大真实捕获而不只依赖本地派生
- `run-radish-docs-qa-real-batch.py` 这条治理入口当前仍较长，后续若继续扩展更多 batch 级治理动作，可能还需要再抽更细的 orchestration helper
- 当前 `real-derived negative index` 已补到 pattern 分组，但 pattern 仍来自 `tags` 约定；后续若派生策略继续增多，可能还要评估是否把这些 pattern 升级成更正式的结构化字段
- 当前 `violation_groups` 已因 action title 的样本差异扩到 9 组；如果后续需要横看“同类 confirmation 漂移”而弱化具体标题影响，可能还要评估是否补一层 violation pattern 归一化字段
